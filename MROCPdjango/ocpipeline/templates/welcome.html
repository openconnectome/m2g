Welcome to the Magnetic Resonance One-Click Pipeline (MROCP) for Connectomes [BETA]

Welcome! This is an Open Source Project that provides Web Services to perform a scientifically interesting tasks involving Connectomes .
Services here can either be accessed programmatically or via web forms.

Want to learn more about the OpenConnectome projects?

Form based use

You may choose to:
  • Upload derivatives (fiber tractography and region of interest files) and build a graph optionally compute invariants.
  • Compute Invariants of graphs built, including:
    • The Scan Statistic 1
    • The Triangle Count
    • The Clustering Coefficient
    • The Maximum Average Degree
    • The Local Degree Count
    • The Top 100 eigenvalues
    • The Scan Statistic 2 [Coming soon]
    • The Average Path Length [Coming soon]
    • The Graph Diameter [Coming soon]
  • Upload a graph previously built through MROCP and compute other desired invariants.
  • Convert data files between npy, mat and csv formats. Data files include built graphs, largest connected component, and all invariants.

Programmatic use

All services can also be accessed programmatically as follows:

  • Upload derivatives to build a graph

    This services base url for an upload is http://mrbrain.cs.jhu.edu/disa/upload/ .
    The example uploadsubj.py can be used as follows from a terminal session:
    python uploadsubj.py http://mrbrain.cs.jhu.edu/disa/upload/{YOUR_project}/{YOUR_site}/{YOUR_subject}/{YOUR_session}/{scanID}/yes|no. The last param is either 'yes' or 'no', but not both: 'yes' adds .mat files for largest connected component and single value decomposition into .mat format , while 'no' does not.
    Multi-subject/scan example is also provided for batch uploading and processing of multiple scans. The use is similar to uploadsubj.py. Help can be obtained by typing in a teminal/command line session: python uploadMultiSubj.py -h , for multi or python uploadsubj.py -h for single scans.

  • Compute Invariants of built graphs

    This is not yet available via programmatic API

  • Convert data file formats

    The use is similar to building a graph progmatically. The base url is http://mrbrain.cs.jhu.edu/disa/convert/.
    Single and batch conversion examples can be found in the code examples.
    An example of executing the sample code from the command line is python singleConvert.py filename.npy http://mrbrain.cs.jhu.edu/disa/convert/cc/mat,csv
    This would convert a file of file type cc from npy format to mat and csv format.
    The following are the ONLY accepted file types:
    • 'ss1' or 'scanStat1' for the Scan Statistic 1
    • 'tri' or 'triangle' for the Triangle Count
    • 'cc' or 'clustCoeff' for the Clustering Coefficient
    • 'mad' or 'maxAvgDeg' for the Maximum Average Degree
    • 'deg' or 'degree' for The Local Degree Count
    • 'eig' or 'eigen' for the Top 100 eigenvalues
    • 'ss2'or 'scanStat2' for the Scan Statistic 2 [Coming soon]
    • 'apl' or 'avePathLen'for the Average Path Length [Coming soon]
    • 'gdia' or 'graphDiam' for the Graph Diameter [Coming soon]
