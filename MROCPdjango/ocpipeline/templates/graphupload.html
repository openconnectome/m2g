Specifications for acceptable input graphs

  • Any igraph supported format is the preferred format for data ingestion.
  • A MATLAB struct with a single data element that is of type numpy CSC (Compressed Sparse Column) matrix.
  • Any 2 Dimensional square MATLAB matrix.
    • It should be loadable from the python shell via the following call: scipy.io.loadmat(filename)['your_data_element_name'] only.
    Such a matrix can be created by creating an MATLAB matrix and using the save function or via Python's scipy.io.savemat function.

Programmatic use

This is can be accessed using the base url http://www.mrbrain.cs.jhu.edu/disa/graphupload/. The example python script can be found here.

How to use the example

After downlaoding the example, start a terminal session. Assuming you are in the same directory as the example. Typing python computeInvariants.py -h will give help on how to use the example and what options are available to you. Below are examples of its use.

Single graph

Assume we have a single graph located in the same directory as the example script named data.mat. Typing python computeInvariants.py http://www.mrbrain.cs.jhu.edu/disa/graphupload/ cc,tri,ss1 data.mat -l -c mat will upload your data.mat file and compute: clustering coefficient, local triangle count, and scan statistic-1; the LCC will be computed and used in invariant computation; the invariants will be converted to .mat format. Here is a breakdown of the command line call arguments:

  1. http://www.mrbrain.cs.jhu.edu/disa/graphupload/ is the base url for uploading graphs for invariants to be computed. This is constant and must not be changed.
  2. cc,tri,ss1 is a comma separated (no spaces) list of the invariants you want computed.
    3. The only available options for invariants are:
    • ss1 for the Scan Statistic 1.
    • tri for the Triangle Count.
    • cc for the Clustering Coefficient.
    • mad for the Maximum Average Degree.
    • deg for The Local Degree Count.
    • eig for the Top-k (50 or maximum available) eigenvalues and eigenvectors.
  3. The -f flag allows you to pass the format of the graph(s) being uploaded. Current available options for graph format are:
    mat, graphml, pajek, ncol, lgl, graphml, dimacs, gml, dot, leda
Starting a tmux or screen session is recommended as you must wait for the http response to otbain a url containing the result.

Multiple graphs

The same example script can be used for multiple graph with or without LCCs. Simply zip all graphs into a single zipfile. If you have precomputed LCCs, zip these into the same zipfile with their corresponding graphs. You must observe the same file naming conventions as described for in-browser use. Other than zipping graphs (and optinal LCCs) together the functionality is identical to single graph use.
