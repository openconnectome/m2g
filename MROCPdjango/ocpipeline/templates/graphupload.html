Specifications for acceptable input graphs

  • Any igraph supported format is the preferred format for data ingestion.
  • A MATLAB struct with a single data element that is of type numpy CSC (Compressed Sparse Column) matrix.
  • Any 2 Dimensional square MATLAB matrix.
    • It should be loadable from the python shell via the following call: scipy.io.loadmat(filename)['your_data_element_name'].
    Such a matrix can be created by creating an MATLAB matrix and using the save function or via Python's scipy.io.savemat function.

Programmatic use

http://www.mrbrain.cs.jhu.edu/disa/graphupload/
The example python script can be found here.

How to use the example

After downlaoding the example, start a terminal session. Assuming you are in the same directory as the example. Typing python computeInvariants.py -h will give help on how to use the example and what options are available to you. Below are examples of its use.

Single graph

Assume we have a single graph located in the same directory as the example script named data.graphml. Typing python computeInvariants.py http://www.mrbrain.cs.jhu.edu/disa/graphupload/ cc,tri,ss1 data.graphml -f graphml will upload your data.graphml file and compute: clustering coefficient, local triangle count, and scan statistic-1. -f flag specifies the format of the graph. If the -f flag is not used then we infer type based on file extension. All legal formats are noted here. Here is a breakdown of the command line call arguments: 

  1. http://www.mrbrain.cs.jhu.edu/disa/graphupload/
    is the base url for uploading graphs for invariants to be computed.
  2. cc,tri,ss1 is a comma separated (no spaces) list of the invariants you want computed.
    3. The only available options for invariants are:
    • ss1 for the Scan Statistic 1.
    • tri for the Triangle Count.
    • cc for the Clustering Coefficient.
    • mad for the Maximum Average Degree.
    • deg for The Local Degree Count.
    • eig for the Top-k (50 or maximum available/computable) eigenvalues and eigenvectors.
  3. The -f flag allows you to pass the format of the graph(s) being uploaded. Current available options for graph format are:
    graphml, ncol, edgelist, lgl, pajek, graphdb, mat, npy
Starting a tmux or screen session is recommended as you must wait for the http response to otbain a url containing the result.

Multiple graphs

The same example script can be used for multiple graphs. Simply zip all graphs into a single zipfile. Other than zipping graphs together the functionality is identical to single graph programmatic use.
