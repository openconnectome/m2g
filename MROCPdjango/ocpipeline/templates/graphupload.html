Please upload the graph(s) with largest connected components(LCC), and ROIs for each graph you would like invariants to be computed.

You must use a single zipfile (.zip) containing ALL data in a single directory even for multiple graphs.
DO NOT zip a directory instead zip the files themselves into an archive. Note the following file naming convention for single graph below where the base file name is the the same:

  • The base file name here is graph1
  • Graph file name: graph1_fiber.mat, graph1_smgr.mat graph1_bggr.mat are acceptible
  • LCC file name: graph1_concomp.mat (Big graphs only)
  • ROI's: graph1_roi.xml and graph1_roi.raw (Big graphs only)
  • All graphs must have their suffix be either _fiber.mat, _smgr.mat or _bggr.mat. Eg graph1_fiber.mat, xyz_smgr.mat and foobar_bggr.mat are all acceptable names.

Specifications for acceptable graphs

  • A MATLAB struct with a data element named named fibergraph that is a numpy CSC (Compressed Sparse Column) matrix.
    It should be loadable with using scipy.io.loadmat(filename)['fibergraph'].

    • What's the difference between big and small graphs?

    We consider small graphs to be 70 x 70 vertices of a CSC matix. The service will accept larger or smaller matrices to be considered as a small graph but we do not recommend it as it may produce unexected be behaviour.
    The big graphs the MROCP constructs are on the order of < 10 M x 10 M vertices, but is other wise identical small graph. Working with big graphs requires the largest connected component (LCC) as well as we do not consider grey matter in evaluation.
